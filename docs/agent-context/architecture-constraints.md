# 架构约束

本文件定义仓库级 AI 编码代理必须尊重的边界。

## 核心原则

- 页面层负责展示与轻量交互，不承载复杂编排。
- 共享协议、访问逻辑、跨页面复用能力应进入 `src/lib/` 或其他共享层。
- 类型定义保持集中，避免在页面里临时拼结构。
- 数据库迁移与边缘函数视为高风险区域。

## 当前仓库的明确约束

### 1. `src/views/` 不堆复杂 agent 编排

- 页面可以展示 Harness 结果
- 页面可以触发共享层调用
- 页面不应成为系统级治理逻辑的主实现位置

### 2. 通用规则优先于工具专属规则

- `AGENTS.md` 与 `docs/agent-context/` 是单一事实来源
- `.cursor/rules/` 只做投影和增强
- 不要在 `.cursor` 里另起一套冲突规范

### 3. 高风险目录优先收敛改动

- 修改 `supabase/migrations/`、`supabase/functions/`、`src/router/` 时，要明确范围与理由
- 避免把无关改动混入高风险提交

### 4. 验证逻辑进入脚本与 CI

- 不把关键质量门槛只写在文档里
- 能自动检查的规则，尽量进入 `scripts/`、hooks 或 CI

## 代理在实现时应避免

- 在页面里复制协议定义
- 为了“快”而跳过共享文档与本地 gates
- 为了凑通过率新增低价值测试
- 无理由重命名、挪动大量文件
