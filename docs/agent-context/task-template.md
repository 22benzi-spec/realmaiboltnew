# 任务模板

任何 AI 编码代理在开始实现前，都应尽量把用户需求整理成这个结构。

## 最小任务结构

- `goal`: 用户真正想达成什么
- `scope`: 允许改哪些文件或模块
- `non_goals`: 明确不做什么
- `risk_level`: `low | medium | high`
- `acceptance`: 什么算完成
- `verification`: 需要运行哪些检查

## 中文模板

```md
## Goal
一句话说明最终目标。

## Scope
- 允许改动：
- 不应改动：

## Non-goals
- 这次不处理什么

## Risk
- 风险等级：
- 风险原因：

## Acceptance
- 用户看到什么结果算完成

## Verification
- 需要执行的命令或检查
```

## 使用原则

- 范围不清时，先缩小范围，不要自由扩散。
- 高风险任务必须明确人工确认点。
- 如果用户只给一句模糊需求，先在内部补齐最小任务结构，再开始实现。
