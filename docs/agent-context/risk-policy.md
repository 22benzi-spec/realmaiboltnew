# 风险策略

本文件定义仓库级 AI 编码任务的风险分级。

## `low`

典型场景：

- 文档更新
- 小型页面文案调整
- 样式修复
- 只读分析
- 不改变行为的轻量重构

默认处理：

- 通过 reviewer 与本地 gates 后可直接交付

## `medium`

典型场景：

- 单模块业务逻辑改动
- 新增页面或组件
- 本地脚本、规则、hook、CI 的局部改动
- 受控范围内的重构

默认处理：

- 必须通过本地 gates
- 必须经过 reviewer

## `high`

典型场景：

- `supabase/migrations/`
- 权限、认证、用户角色
- 财务、退款、金额逻辑
- 删除操作或不可逆行为
- 跨层依赖调整
- 大范围重构

默认处理：

- 必须通过本地 gates 与 CI
- 必须经过 reviewer
- 默认需要人工确认

## 高风险目录提示

以下目录或文件一旦改动，应优先视为高风险候选：

- `supabase/migrations/`
- `supabase/functions/`
- `src/router/`
- `src/lib/supabase.ts`
- `src/views/` 中涉及资金、权限、用户、订单状态流转的页面

## 输出要求

交付时必须明确：

- 当前风险等级
- 风险原因
- 已执行的验证
- 是否需要人工确认
